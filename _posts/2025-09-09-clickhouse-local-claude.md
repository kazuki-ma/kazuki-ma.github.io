## はじめに

Claude Code を使い始めて数ヶ月、気がつけば大量の会話ログが溜まっていました。`ccusage` コマンドで基本的な使用状況は確認できますが、もっと詳細な分析がしたい。そこで ClickHouse Local を使って、Claude の使用ログを分析するスクリプトを作成しました。

## Claude Code のログ構造

Claude Code は `~/.claude/projects/` ディレクトリ以下に、プロジェクトごとに JSONL（JSON Lines）形式でログを保存しています：

```bash
~/.claude/projects/
├── project-name-1/
│   ├── session-uuid-1.jsonl
│   ├── session-uuid-2.jsonl
│   └── ...
└── project-name-2/
    └── ...
```

各 JSONL ファイルには、セッションごとのメッセージ、ツール使用状況、タイムスタンプなどが記録されています。

## ClickHouse Local とは

ClickHouse Local は、ClickHouse データベースのスタンドアロン版で、サーバーを起動することなくローカルファイルに対して SQL クエリを実行できます。特に JSON、CSV、Parquet などの構造化データの分析に優れています。

### インストール

macOS の場合：
```bash
brew install clickhouse
```

## 分析から得られた知見

1. **使用パターン**: 早朝と昼間に集中的に使用している
2. **プロジェクト分布**: 特定のプロジェクトに使用が集中している
3. **セッション長**: 最長セッションは1,731メッセージ、27時間以上継続
4. **週末 vs 平日**: 意外にも週末と平日で使用量に大きな差がない

## まとめ

ClickHouse Local を使うことで、複雑な集計処理を SQL で簡潔に記述でき、大量のログファイルも高速に処理できました。Claude Code のようなツールの使用状況を定量的に把握することで、自分の開発パターンを客観的に理解できます。

このスクリプトは GitHub で公開していますので、ぜひご自身の Claude 使用状況の分析にお役立てください。
